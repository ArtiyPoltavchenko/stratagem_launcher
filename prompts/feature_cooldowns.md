# Feature: Ship Module Cooldown Modifier

## Контекст
Кулдаун-таймеры на карточках стратагем уже работают (Phase 4) — после нажатия идёт обратный отсчёт, блокирующий повторный тап. Базовые значения cooldown берутся из stratagems.json.

Однако в Helldivers 2 кулдауны уменьшаются при улучшении модулей корабля (Ship Module upgrades). Также бывают глобальные события, временно срезающие кулдауны. Сейчас нет способа это отразить в PWA.

## Что нужно (фронтенд-только, сервер не трогаем)

### 1. Глобальный множитель кулдауна в Settings
В существующей секции настроек (`web/app.js`, `web/index.html`), ПОСЛЕ ползунка Key delay:

```
Key delay:           [====●=========] 50ms  [50]
Cooldown reduction:  [====●=========] 0%    [0 ]
☑ Show cooldowns on cards
```

- **Range slider**: 0% — 50%, шаг 5%. По умолчанию 0% (без скидки)
- **Input number** рядом — синхронизировано с ползунком
- При изменении — мгновенно пересчитать все кулдауны на карточках
- Значение сохраняется в localStorage (ключ `sl_cooldown_modifier`)

### 2. Пересчёт кулдаунов
Существующий cooldown timer overlay уже показывает обратный отсчёт. Нужно изменить базовое значение:

```javascript
// Текущее поведение (Phase 4):
const cooldown = stratagem.cooldown; // из JSON

// Новое поведение:
const modifier = parseInt(localStorage.getItem('sl_cooldown_modifier') || '0');
const cooldown = Math.round(stratagem.cooldown * (1 - modifier / 100));
// modifier=0  → 100s * 1.0  = 100s (default)
// modifier=25 → 100s * 0.75 = 75s
// modifier=50 → 100s * 0.5  = 50s
```

Применить пересчёт везде, где используется `stratagem.cooldown`:
- При запуске таймера после нажатия
- В отображении кулдауна под иконкой (если показывается)

### 3. Toggle: Show/Hide cooldowns
- Чекбокс "Show cooldowns on cards"
- По умолчанию: включено
- Скрывает: текст кулдауна под иконкой + таймер-оверлей после нажатия
- Когда выключено — карточки нажимаются без блокировки (fire-and-forget)
- Сохранять в localStorage (ключ `sl_show_cooldowns`)

### 4. Отображение модифицированного кулдауна
Под иконкой стратагемы — если показывается кулдаун и modifier > 0:
- Показать зачёркнутое старое + новое: `~~100s~~ 75s` или просто `75s` цветом
- Цвет текста: приглушённый серый (`var(--text-secondary)`)

## Что НЕ трогаем
- `server/` — ничего, это чисто UI фича
- `data/stratagems.json` — базовые cooldown значения остаются
- Существующий таймер-оверлей — не переписывать, только подставить модифицированное значение
- Loadout view (Phase 6) — кулдауны там тоже должны пересчитываться

## Тестирование
- Ручное: открыть Settings, подвигать ползунок, проверить что таймеры поменялись
- Убедиться что localStorage сохраняет/восстанавливает при перезагрузке
- 67 существующих тестов должны проходить без изменений
